Chapter 1 – Introduction 
1.1 Project Overview 
DockWatch is a DevOps-based monitoring dashboard developed to monitor Docker containers 
and system resources through a centralized web interface. The project follows a microservices 
architecture and integrates modern DevOps tools such as Docker, Docker Compose, Jenkins, 
GitHub Actions, and GitHub. 
The dashboard allows users to view container status, CPU utilization, memory utilization, and 
container logs. The project also demonstrates Continuous Integration (CI) and Continuous 
Deployment (CD) using GitHub Actions and Jenkins. 
 
1.2 Problem Statement 
Managing multiple Docker containers manually can become difficult as the number of services 
increases. Developers often need to execute several commands to monitor container status, view 
logs, and check resource utilization. 
Traditional monitoring approaches require multiple tools and manual intervention. Additionally, 
deployment and updates can become time-consuming without automation. 
Therefore, a centralized monitoring platform integrated with DevOps practices is required to 
simplify monitoring and automate deployment workflows. 
 
1.3 Objectives 
The objectives of the project are: 
• To develop a centralized monitoring dashboard. 
• To monitor Docker containers in real time. 
• To display CPU and RAM utilization. 
• To implement microservices architecture. 
• To automate deployment using Jenkins. 
• To implement CI using GitHub Actions. 
• To demonstrate containerization using Docker. 
 
1.4 Scope 
The project can be used by developers and system administrators to monitor containerized 
applications. 
Scope of the project includes:

The project workflow is illustrated below: 
1. Developer writes code. 
2. Code is pushed to GitHub repository. 
3. GitHub Actions automatically validates the project.
